设计模式初探（三），单例工厂模式

2019-04-02 /
  • 设计模式
 / 点击 /

  今天正式开始标准的23种设计模式的讲解，将设计模式按照功能划分，
大体可以分为如下三种：
  1、创建型设计模式
  2、结构型设计模式
  3、行为型设计模式
  单例工厂设计模式是创建型设计模式，负责对象的创建,单例顾名思义，是指只能创建一个对象，在java中创建对象一般使用new关键字，例如：


  Student student = new Student();

那么如何控制一个类只能创建一个对象呢？那么首先就需要将类的构造器私有化，让外部类无法创建对象，这样就控制了类的创建


public class Student{
    private Student(){
    }
}

但是还需要提供一个对象给外部类使用


public class Student{
    private static Student student = new Student();
    private Student(){
    }
    public static Student getInstance(){
        return student;
    }
}

至此我们就实现了单例模式
下面介绍几款单例模式的经典实现：

饿汉：

顾名思义，就是不管需不需要我都先创建对象，然后直接调用就行


public class Student{
    private static Student student = new Student();
    private Student(){
    }
    public static Student getInstance(){
        return student;
    }
}

懒汉 1 线程不安全:

就是等到真正需要用到的时候，再创建对象


public class Student{
    private static Student student = null;
    private Student(){
    }
    public static Student getInstance(){
        if(student==null){
            student = new Student();
        }
        return student;
    }
}

懒汉 2 线程安全:

懒汉1没有做线程保护，在多线程环境中并不安全，懒汉2加入了线程保护


public class Student{
    private static Student student = null;
    private Student(){
    }
    public synchronized static Student getInstance(){
        if(student==null){
            student = new Student();
        }
        return student;
    }
}

懒汉 3 双重检查加锁:

懒汉2的加锁方式对性能的损耗过大，双加锁的方式极大的改善了此问题


public class Student{
    private volatile static Student student = null;
    private Student(){
    }
    public static Student getInstance(){
        if(student==null){
            synchronized(Student.class){
                if(student==null){
                    student = new Student();
                }
            }
        }
        return student;
    }
}

静态内部类:

因为jvm提供了静态属性的线程安全，所以可以使用静态内部类来实现，线程安全性能高的饿汉模式


public class Student{
    private Student(){
    }
    public static Student getInstance(){
        return StudentHolder.instance;
    }
    public static Class StudentHolder(){
        private static Student instance = new Student();
    }
}

但是值得一提的是，所谓单例也是针对单jvm下的单类加载器，在多个类加载器中肯定都能存在一个单例模式的对象！
